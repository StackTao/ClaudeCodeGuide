### 2.1.154（2026-05-28）

**新功能**
- Opus 4.8 发布！现在默认使用 high effort；`/effort xhigh` 用于你最难的任务。
- 推出 dynamic workflows（动态工作流）：让 Claude 创建一个工作流，它会在后台跨数十到数百个 agent 编排工作，让你能承担更大、更复杂的任务。运行 `/workflows` 查看你的运行记录。
- Opus 4.8 上的 fast mode 现在以远低于此前的成本提供：2 倍标准费率换取 2.5 倍速度。
- `claude agents`：输入 `! <command>` 可把一条 shell 命令作为可 attach / detach 的后台会话运行；也可用 `claude --bg --exec '<command>'`。
- Plugins 现在可以在 `plugin.json` 或 marketplace 条目中声明 `defaultEnabled: false`；用 `/plugin` 或 `claude plugin enable` 启用。已启用插件的依赖仍会自动启用。
- 为 `/claude-api` skill 增加了 Claude Opus 4.8 支持以及 4.7 → 4.8 的迁移指引。

**改进**
- 现在除 Haiku、Sonnet 以及 Opus 4.7 及更早的模型外，精简版系统提示（lean system prompt）成为默认。
- Claude 现在只在它确实无法自行决定时才弹出多选问题，而不是在已有足够上下文可继续时发问。
- `/simplify` 现在运行仅清理的审查（复用、简化、效率、抽象层级 altitude）并应用修复，而不再运行完整的 `/code-review --fix` 找 bug 审查。
- 把 `/effort` 滑块标签从 “Speed”/“Intelligence” 改名为 “Faster”/“Smarter” 以更清晰。
- `claude agents`：`/logout` 现在会让你登出，而不是被送进一个后台会话。
- `←←` 打开 agents 视图现在在 Bedrock、Vertex、Foundry 以及关闭遥测时都可用。
- Claude in Chrome：通过 `/chrome` →「Select browser…」选择要使用的已连接浏览器；当某个浏览器操作在多个已连接浏览器下运行时也可在对话内选择。
- `/plugin` Discover 标签页现在会把与当前目录相关性信号匹配的插件置顶，并标注「suggested for this directory」。
- 流式工具执行（streaming tool execution）现在始终启用，包括关闭遥测时或在 Bedrock/Vertex/Foundry 上（此前在功能开关后面）。
- Stdio MCP server 子进程现在在其环境中会收到 `CLAUDE_CODE_SESSION_ID` 和 `CLAUDECODE=1`。
- `claude mcp list`/`get` 在输出被管道传递时，现在把未批准的 `.mcp.json` server 显示为 `⏸ Pending approval`，而不是自动批准并连接。
- `/remote-control` 自动补全现在在 Remote Control 已激活时显示「Disconnect Remote Control」。
- 弃用 `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE`（将于 06/01 移除）。要在 Opus 4.6 上使用 fast mode，请用 `/model claude-opus-4-6[1m]` 切换后再 `/fast on`。
- 从快捷键帮助面板移除了过时的「& for background」提示。
- [VSCode] Auto mode 不再需要 bypass-permissions 设置出现在模式选择器中；新会话界面上有一个可关闭的提示，会在首次启用 auto mode 时进行说明。

**安全**
- 改进了 auto-mode 分类器对数据外泄（data exfiltration）的检测，尤其是仓库内容的批量传输。
- 修复 `HOME` 带尾部斜杠时 `rm -rf $HOME` 未被作为危险路径拦截的问题。

**修复**

### 2.1.152（2026-05-27）

**新功能**
- 新增 `/reload-skills` 命令，无需重启会话即可重新扫描 skill 目录。
- 新增 `MessageDisplay` hook 事件，允许 hook 在助手消息文本显示时对其进行改写或隐藏。
- 新增 `pluginSuggestionMarketplaces` 托管设置：管理员可允许列出哪些组织 marketplace 的插件可通过上下文相关提示被推荐。

**改进**
- `/code-review --fix` 现在会在审查后把审查结论应用到你的工作区，并给出复用、简化和效率方面的建议；`/simplify` 现在会调用 `/code-review --fix`。
- Skills 和 slash 命令现在可以在 frontmatter 中设置 `disallowed-tools`，在该 skill 激活期间从模型移除这些工具。
- `SessionStart` hook 现在可以返回 `reloadSkills: true` 来重新扫描 skill 目录，使该 hook 安装的 skill 在同一会话内即可用。
- `SessionStart` hook 现在可以在启动和 resume 时通过 `hookSpecificOutput.sessionTitle` 设置会话标题。
- `claude plugin marketplace remove` 现在接受 `--scope user|project|local`，与 `marketplace add`、`install`、`uninstall` 对称。
- 当主模型未找到时，Claude Code 现在会在本会话剩余时间切换到你配置的 `--fallback-model`，而不是让每个请求都失败。
- Auto mode 不再需要事先 opt-in 同意。
- Vim 模式：NORMAL 模式下的 `/` 现在打开反向历史搜索（类似 Ctrl+R），与 bash/zsh 的 vi-mode 一致。
- `/usage` 的明细现在包含大型会话文件；这些文件以流式读取扫描，使内存占用保持平稳。
- 折叠分组中的 thinking 摘要现在至少保持 3 秒可读、按 markdown 渲染并最多 10 行（`Ctrl+O` 显示完整 thinking）。
- 全屏模式下，「Thinking for Ns」指示器现在会在模型思考时实时计数，并在你中途打断时保留其值。
- 简化了 Workflow 工具的内联进度显示——实时 agent 计数现在只显示在提示下方常驻的 workflow 状态行中。
- 响应后的计时器现在在后台 agent 或 workflow 仍在运行时显示「Waiting for N background agents/workflows to finish」，并在其结果处理完后报告累计时间。
- 把会话入口（session entrypoint）作为 OpenTelemetry 指标属性加入（`app.entrypoint`，通过 `OTEL_METRICS_INCLUDE_ENTRYPOINT=true` 选用）。

**修复**

- 修复 sandbox-enabled 警告在精简启动模式下不显示的问题——现在在每种布局下都会显示。
- 修复工具运行时加载 spinner 仍显示「still thinking」/「almost done thinking」的问题，并在每次工具调用后把 thinking 状态重置为「thinking」。

- 修复在使用 `--bare` 或禁用附件运行时，Agent 工具描述引用一个永远不会送达的 agent 列表的问题。

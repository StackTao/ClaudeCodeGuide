## v0.x 系列（0.2.125 – 0.2.21）

### 0.2.125

**改进**
- 重大变更：传给 `ANTHROPIC_MODEL` 或 `ANTHROPIC_SMALL_FAST_MODEL` 的 Bedrock ARN 不应再包含经过转义的斜杠（请使用 `/` 而非 `%2F`）
- 移除了 `DEBUG=true`，改用 `ANTHROPIC_LOG=debug` 来记录所有请求

### 0.2.117

**新功能**
- 引入了 `settings.cleanupPeriodDays`
- 引入了 `CLAUDE_CODE_API_KEY_HELPER_TTL_MS` 环境变量
- 引入了 `--debug` 模式

**改进**
- 重大变更：`--print` 的 JSON 输出现在返回嵌套的 message 对象，以便在我们引入新的元数据字段时保持向前兼容

### 0.2.108

**新功能**
- 现在你可以在 Claude 工作时向其发送消息，从而实时引导 Claude
- 引入了 `BASH_DEFAULT_TIMEOUT_MS` 和 `BASH_MAX_TIMEOUT_MS` 环境变量

**改进**
- 弃用了 MCP wizard 界面，改用其他 MCP 命令
- 大量其他的错误修复与改进

**修复**
- 修复了在 `-p` 模式下 thinking 无法工作的问题
- 修复了 `/cost` 报告中的一处回归

### 0.2.107

**新功能**
- `CLAUDE.md` 文件现在可以导入其他文件。在 `./CLAUDE.md` 中添加 `@path/to/file.md`，即可在启动时加载额外的文件

### 0.2.106

**新功能**
- MCP SSE 服务器配置现在可以指定自定义请求头

**修复**
- 修复了 MCP 权限提示并非总能正确显示的问题

### 0.2.105

**新功能**
- Claude 现在可以搜索网络

**改进**
- 将系统与账户状态移至 `/status`
- 为 Vim 添加了按词移动的快捷键
- 改进了启动、todo 工具和文件编辑的延迟

### 0.2.102

**改进**
- 提升了 thinking 触发的可靠性
- 提升了图片和文件夹 `@mention` 的可靠性
- 现在你可以将多个大段内容粘贴到一个 prompt 中

### 0.2.100

**改进**
- 将数据库存储设为可选；缺少数据库支持会禁用 `--continue` 和 `--resume`

**修复**
- 修复了由栈溢出错误导致的崩溃

### 0.2.98

**修复**
- 修复了 auto-compact 会运行两次的问题

### 0.2.96

**新功能**
- Claude Code 现在也可以配合 Claude Max 订阅使用（https://claude.ai/upgrade）

### 0.2.93

**新功能**
- 使用 `claude --continue` 和 `claude --resume` 从上次中断的地方恢复对话
- Claude 现在可以使用一个 Todo 列表，帮助其保持专注并更有条理

### 0.2.82

**新功能**
- 添加了对 `--disallowedTools` 的支持

**改进**
- 为保持一致性重命名了工具：`LSTool` -> `LS`、`View` -> `Read` 等

### 0.2.75

**新功能**

**改进**
- 在 Claude 工作时按 Enter 可排队添加更多消息
- 将图片文件直接拖入或复制/粘贴到 prompt 中
- 通过 `@-mention` 文件可将其直接添加到上下文中
- 改进了文件名自动补全的性能

### 0.2.74

### 0.2.72

### 0.2.70

### 0.2.69

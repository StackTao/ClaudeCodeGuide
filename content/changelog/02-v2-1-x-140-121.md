## v2.1.x 系列（2.1.140 – 2.1.121）

### 2.1.140

**改进**
- 改进了 Agent 工具的 `subagent_type` 匹配，现可接受大小写与分隔符不敏感的值（例如 `"Code Reviewer"` 会解析为 `code-reviewer`）
- 更新了 agent 颜色调色板
- 当设置了 `disableAllHooks` 或 `allowManagedHooksOnly` 时，插件现会在某个默认组件文件夹（例如 `commands/`）因 `plugin.json` 设置了对应键而被静默忽略时发出警告。该警告会显示在 `/doctor`、`claude plugin list` 和 `/plugin` 中。

**修复**
- 修复了当设置了 `disableAllHooks` 或 `allowManagedHooksOnly` 时 `/goal` 静默挂起的问题——现在会显示一条清晰的提示，而不是一个永远不会结束的指示器
- 修复了设置热重载中的一处回归：软链接的设置文件会导致变更事件被错误归因并产生虚假的 `ConfigChange` 钩子
- 修复了当后台服务即将因空闲而退出时，`claude --bg` 以 "connection dropped mid-request" 失败的问题
- 修复了在装有企业级终端安全软件的机器上后台服务启动失败的问题，方法是给予更多启动时间
- 修复了远程托管设置在遇到 401 时不重试的问题——现在会用强制刷新的令牌重试一次
- 修复了托管的 `extraKnownMarketplaces` 自动更新策略未被持久化到 `known_marketplaces.json` 的问题
- 修复了 `/loop` 为那些已会在完成时通知的后台任务安排冗余唤醒以进行轮询的问题
- 修复了 Windows 上一处反复出现的事件循环停顿：当缺少某个可执行文件（例如 `gh`）时，每次检查都会触发同步的 `where.exe` 重新派生
- 修复了当 `offset` 以带空白填充或带 `+` 前缀的字符串形式传入时 `Read` 工具调用校验失败的问题
- 修复了终端失去焦点时原生终端光标未停留在输入插入符处的问题

### 2.1.139

**新功能**
- 新增 agent 视图（研究预览）：将每一个 Claude Code 会话——正在运行、等待你处理、或已完成的——汇总为单个列表。运行 `claude agents` 即可开始。参见 https://code.claude.com/docs/en/agent-view
- 新增 `/goal` 命令：设定一个完成条件，Claude 会跨多个回合持续工作，直到该条件被满足。支持交互模式、`-p` 和 Remote Control。会以浮层面板形式显示实时的耗时／回合数／token 数
- 新增 `/scroll-speed` 命令，可通过实时预览调节鼠标滚轮的滚动速度
- 新增 `claude plugin details <name>`，用于显示某个插件的组件清单及其预计的单会话 token 成本
- 新增转录视图导航：`?` 查看键盘快捷键，`{`/`}` 在用户提示之间跳转，`v` 切换快捷键面板
- 新增钩子的 `args: string[]` 字段（exec 形式），它会直接派生命令而不经过 shell，因此路径占位符无需再加引号
- 为 `PostToolUse` 新增钩子的 `continueOnBlock` 配置选项——设为 `true` 可将钩子的拒绝原因反馈给 Claude 并继续当前回合

**改进**
- MCP stdio 服务器现在会在其环境中收到 `CLAUDE_PROJECT_DIR`，与钩子保持一致。插件配置可在命令中引用 `${CLAUDE_PROJECT_DIR}`
- 压缩（compaction）提示现在会要求模型保留敏感的用户指令
- `/mcp` 的 Reconnect 现在无需重启即可读取 `.mcp.json` 的编辑，并在重连失败时显示 HTTP 状态码和 URL
- `/context all` 的每个 skill 的 token 估算现在会考虑模型的分词器并显示取整后的值
- `claude plugin install <name>@<marketplace>` 现在会先自动刷新市场并重试，然后才报告找不到该插件
- `/plugin` 已安装插件的详情现在会清晰地显示钩子事件名和 MCP 服务器名
- `/context` 现在会为来自插件的 skill 显示提供它的插件名
- 远程 MCP 服务器在遇到瞬时故障时的重连重试现在对所有用户启用
- 来自子代理的 API 请求现在会携带 `x-claude-code-agent-id` / `x-claude-code-parent-agent-id` 头，并且 `claude_code.llm_request` OTEL span 包含 `agent_id` / `parent_agent_id` 属性
- 当设置了 `ANTHROPIC_API_KEY` / `apiKeyHelper` / `ANTHROPIC_AUTH_TOKEN` 时，即使同时存在 Claude.ai 登录，Remote Control、`/schedule`、claude.ai MCP 连接器和通知偏好现在也会被禁用。取消设置该 API key 即可使用这些功能

**改进**
- 全屏模式：在你向上滚动以阅读较早输出后，向提示中键入不再将滚动跳回底部
- 溢出终端的对话框现在可在全屏和非全屏模式下使用方向键、PgUp/PgDn、home/end 和鼠标滚轮滚动
- 在全屏模式下点击一个跨行换行的长 URL 的任意一行现在会打开完整 URL
- SDK 和 `claude -p`：`CLAUDE_CODE_FORK_SUBAGENT=1` 现在可在非交互式会话中工作
- `--dangerously-skip-permissions` 不再对写入 `.claude/skills/`、`.claude/agents/` 和 `.claude/commands/` 提示
- `/terminal-setup` 现在会启用 iTerm2 的 "Applications in terminal may access clipboard" 设置，以便 `/copy` 可用，包括从 tmux 中使用
- 在启动时遇到瞬时错误的 MCP 服务器现在会自动重试至多 3 次，而不是保持断开
- 终端标签页会话标题现在会以你配置的 `language` 设置生成
- 具有相同上游 URL 的 Claude.ai 连接器现在会被去重，而不是显示为重复项
- Vertex AI：支持基于 X.509 证书的工作负载身份联合（mTLS ADC）
- 升级后更快启动：从发布说明启动页移除了 Recent Activity 面板
- LSP 诊断摘要现在可在点击/ctrl+o 时展开，并显示展开提示
- SDK：`mcp_authenticate` 现在支持用于自定义方案完成和 claude.ai 连接器的 `redirectUri`
- OpenTelemetry：向 LLM 请求 span 新增 `stop_reason`、`gen_ai.response.finish_reasons` 和 `user_system_prompt`（受 `OTEL_LOG_USER_PROMPTS` 限制）
- [VSCode] 当未配置 Claude Code 语言时，语音听写现在会遵循 `accessibility.voice.speechLanguage` 设置
- [VSCode] `/context` 现在会打开一个原生的 token 用量对话框

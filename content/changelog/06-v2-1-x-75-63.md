## v2.1.x 系列（2.1.75 – 2.1.63）

### 2.1.75

**新功能**
- 为 Max、Team 和 Enterprise plan 默认启用 Opus 4.6 的 1M 上下文窗口（此前需额外用量）。
- 为所有用户新增 `/color` 命令以设置会话提示栏颜色。
- 使用 `/rename` 时在提示栏显示会话名。
- 为 memory 文件新增最后修改时间戳，帮助 Claude 推断哪些 memory 是新鲜还是过期。
- hook 需确认时在权限提示中新增 hook 来源显示（settings/plugin/skill）。

**改进**
- 通过跳过不必要的子进程生成，改进 macOS 非 MDM 机器上的启动性能。
- 默认抑制异步 hook 完成消息（用 `--verbose` 或 transcript 模式可见）。

**修复**
- 修复全新安装上不切换 `/voice` 两次语音模式不正确激活的问题。
- 修复用 `/model` 或 Option+P 切换模型后 Claude Code 头部不更新显示模型名的问题。
- 修复附件消息计算返回 undefined 值时的会话崩溃。
- 修复 Bash 工具在管道命令中破坏 `!` 的问题（如 `jq 'select(.x != .y)'` 现在正常工作）。
- 修复 managed-disabled 插件出现在 `/plugin` Installed 标签的问题——被组织强制禁用的插件现在被隐藏。
- 修复 token 估算对 thinking 和 `tool_use` 块多计、导致过早上下文压缩的问题。
- 修复损坏的 marketplace 配置路径处理。
- 修复恢复 fork 或继续会话后 `/resume` 丢失会话名的问题。
- 修复访问 Config 标签后 Esc 不关闭 `/status` 对话框的问题。
- 修复接受或拒绝 plan 时的输入处理。
- 修复 agent team 中页脚提示显示「↓ to expand」而非正确的「shift + ↓ to expand」的问题。

**其他**
- 破坏性变更：移除已弃用的 Windows managed 设置回退路径 `C:\ProgramData\ClaudeCode\managed-settings.json`——改用 `C:\Program Files\ClaudeCode\managed-settings.json`。

### 2.1.74

**新功能**

- 新增 `autoMemoryDirectory` 设置以为 auto-memory 存储配置自定义目录。

**改进**
- 把 `--plugin-dir` 改为本地 dev 副本现在覆盖同名已安装 marketplace 插件（除非该插件被 managed 设置强制启用）。
- [VSCode] 改进集成终端中带终端感知加速的滚轮响应性。

**修复**
- 修复生成器提前终止时流式 API 响应缓冲未释放、导致 Node.js/npm 代码路径上 RSS 无界增长的内存泄漏。
- 修复 managed 策略 `ask` 规则被用户 `allow` 规则或 skill `allowed-tools` 绕过的问题。
- 修复完整模型 ID（如 `claude-opus-4-5`）在 agent frontmatter `model:` 字段和 `--agents` JSON 配置中被静默忽略的问题——agent 现在接受与 `--model` 相同的模型值。

- 修复退出时 `SessionEnd` hook 无视 `hook.timeout` 在 1.5 s 后被杀的问题——现在可通过 `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` 配置。

- 修复 marketplace 更新不同步 git submodule 的问题——submodule 中的插件源更新后不再损坏。
- 修复带参数的未知 slash 命令静默丢弃输入的问题——现在把你的输入显示为警告。
- 修复希伯来语、阿拉伯语等 RTL 文本在 Windows Terminal、conhost 和 VS Code 集成终端中渲染不正确的问题。
- 修复因畸形文件 URI 导致 LSP server 在 Windows 上不工作的问题。
- [VSCode] 修复 Untitled 会话删除按钮不工作的问题。

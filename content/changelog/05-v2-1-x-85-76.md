## v2.1.x 系列（2.1.85 – 2.1.76）

### 2.1.85

**新功能**
- 为 MCP `headersHelper` 脚本新增 `CLAUDE_CODE_MCP_SERVER_NAME` 和 `CLAUDE_CODE_MCP_SERVER_URL` 环境变量，使一个 helper 可服务多个 server。
- 为 hook 新增条件 `if` 字段，使用权限规则语法（如 `Bash(git *)`）筛选触发时机，减少进程生成开销。
- 计划任务（`/loop`、`CronCreate`）触发时在 transcript 中新增时间戳标记。
- 粘贴图片时在 `[Image #N]` 占位符后新增尾随空格。
- 深链接查询（`claude-cli://open?q=…`）现在支持最多 5,000 字符，长预填提示会带「scroll to review」警告。
- MCP OAuth 现在遵循 RFC 9728 Protected Resource Metadata discovery 来查找授权服务器。
- 被组织策略（`managed-settings.json`）阻止的插件不再能被安装或启用，并从 marketplace 视图中隐藏。
- PreToolUse hook 现在可通过在 `permissionDecision: "allow"` 之外返回 `updatedInput` 来满足 `AskUserQuestion`，使无头集成能用自己的 UI 收集答案。
- OpenTelemetry tool_result 事件中的 `tool_parameters` 现在需 `OTEL_LOG_TOOL_DETAILS=1` 才输出。

**改进**
- 改进大型仓库上的 @-mention 文件自动补全性能。
- 改进 PowerShell 危险命令检测。
- 通过用纯 TypeScript 实现替换 WASM yoga-layout，改进大 transcript 的滚动性能。
- 减少大会话触发压缩时的 UI 卡顿。

**修复**
- 修复对话过大、连 compact 请求本身都放不下时 `/compact` 以「context exceeded」失败的问题。
- 修复插件安装位置与 settings 中声明位置不同时 `/plugin enable` 和 `/plugin disable` 失败的问题。
- 修复 `--worktree` 在非 git 仓库中于 `WorktreeCreate` hook 运行前就报错退出的问题。
- 修复 `deniedMcpServers` 设置未阻止 claude.ai MCP server 的问题。
- 修复多显示器设置上 computer-use 工具 `switch_display` 返回「not available in this session」的问题。
- 修复 `OTEL_LOGS_EXPORTER`、`OTEL_METRICS_EXPORTER` 或 `OTEL_TRACES_EXPORTER` 设为 `none` 时的崩溃。
- 修复非原生构建中 diff 语法高亮失效的问题。
- 修复存在 refresh token 时 MCP step-up 授权失败的问题——server 通过 `403 insufficient_scope` 请求提升权限时现在能正确触发重新授权流程。
- 修复流式响应被打断时远程会话的内存泄漏。
- 修复边缘连接churn 期间持续 ECONNRESET 错误的问题——重试时改用全新 TCP 连接。
- 修复运行某些 slash 命令后提示卡在队列、上方向键无法取回的问题。
- 修复 Python Agent SDK：通过 `--mcp-config` 传入的 `type:'sdk'` MCP server 不再在启动时被丢弃。
- 修复在 SSH 或 VS Code 集成终端中运行时原始按键序列出现在提示中的问题。
- 修复权限解决后 Remote Control 会话状态卡在「Requires Action」的问题。
- 修复 shift+enter 和 meta+enter 被 typeahead 建议拦截而非插入换行的问题。
- 修复流式期间向上滚动时过期内容透出的问题。
- 修复退出后在 Ghostty、Kitty、WezTerm 等支持 Kitty 键盘协议的终端中留下增强键盘模式的问题——退出后 Ctrl+C 和 Ctrl+D 现在正常工作。

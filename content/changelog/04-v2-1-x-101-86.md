## v2.1.x 系列（2.1.101 – 2.1.86）

### 2.1.101

**新功能**
- 新增 `/team-onboarding` 命令，从你本地的 Claude Code 使用情况生成 teammate 上手指南。
- 默认信任 OS CA 证书库，使企业 TLS 代理无需额外设置即可工作（设 `CLAUDE_CODE_CERT_STORE=bundled` 仅用打包 CA）。

**改进**
- `/ultraplan` 和其他远程会话功能现在自动创建默认云环境，而不再要求先做网页设置。
- 改进 brief 模式：当 Claude 以纯文本而非结构化消息响应时重试一次。
- 改进 focus 模式：Claude 现在写更自包含的摘要，因为它知道你只看到它的最终消息。
- 改进工具不可用错误：当模型调用一个存在但当前上下文不可用的工具时，解释原因和如何继续。
- 改进速率限制重试消息，显示触及了哪个限制及何时重置，而非不透明的秒数倒计时。
- 改进拒绝错误消息，在可用时包含 API 提供的解释。
- 改进 `claude -p --resume <name>`，接受通过 `/rename` 或 `--name` 设置的会话标题。
- 改进设置韧性：`settings.json` 中无法识别的 hook 事件名不再导致整个文件被忽略。
- 改进由 managed settings 强制启用的插件 hook，使其在设置 `allowManagedHooksOnly` 时运行。
- 改进 `/plugin` 和 `claude plugin update`，在 marketplace 无法刷新时显示警告，而非静默报告过期版本。
- 改进 plan 模式，当用户组织或鉴权设置无法访问 Claude Code on the web 时隐藏「Refine with Ultraplan」选项。
- 改进 beta tracing，遵循 `OTEL_LOG_USER_PROMPTS`、`OTEL_LOG_TOOL_DETAILS` 和 `OTEL_LOG_TOOL_CONTENT`；除非选用，敏感 span 属性不再输出。
- 改进 SDK `query()`，在消费者从 `for await` `break` 或使用 `await using` 时清理子进程和临时文件。

**安全**
- 修复 LSP 二进制检测所用 POSIX `which` 回退中的命令注入漏洞。
- 修复 `permissions.deny` 规则未覆盖 PreToolUse hook 的 `permissionDecision: "ask"` 的问题——此前 hook 可把 deny 降级为提示。

**修复**
- 修复长会话在虚拟滚动器中保留消息列表数十个历史副本的内存泄漏。
- 修复加载器锚定在死分支而非活动对话时 `--resume`/`--continue` 在大会话上丢失对话上下文的问题。
- 修复当 subagent 消息落在主链写入间隙附近时 `--resume` 链恢复桥接进无关 subagent 对话的问题。
- 修复持久化的 Edit/Write 工具结果缺少 `file_path` 时 `--resume` 崩溃的问题。
- 修复一个硬编码 5 分钟请求超时，它无视 `API_TIMEOUT_MS` 中止慢后端（本地 LLM、扩展思考、慢 gateway）的问题。
- 修复缺少 `user` 的 `--setting-sources` 导致后台清理忽略 `cleanupPeriodDays` 并删除超过 30 天对话历史的问题。
- 修复当 `ANTHROPIC_AUTH_TOKEN`、`apiKeyHelper` 或 `ANTHROPIC_CUSTOM_HEADERS` 设置了 Authorization header 时 Bedrock SigV4 认证以 403 失败的问题。
- 修复先前会话 worktree 清理留下过期目录后 `claude -w <name>` 以「already exists」失败的问题。
- 修复 subagent 不继承动态注入 server 的 MCP 工具的问题。
- 修复在隔离 worktree 中运行的 subagent 被拒绝读/编辑其自身 worktree 内文件的问题。
- 修复全新启动后沙箱 Bash 命令以 `mktemp: No such file or directory` 失败的问题。
- 修复在校验 `outputSchema` 的 MCP 客户端中 `claude mcp serve` 工具调用以「Tool execution failed」失败的问题。
- 修复 `RemoteTrigger` 工具 `run` 动作发送空 body 而被 server 拒绝的问题。
- 修复多个 `/resume` 选择器问题：窄默认视图隐藏其他项目会话、Windows Terminal 上预览不可达、worktree 中 cwd 不正确、session-not-found 错误不在 stderr 浮现、终端标题未设置、resume 提示与提示输入重叠。
- 修复内嵌 ripgrep 二进制路径过期（VS Code 扩展自动更新、macOS App Translocation）时 Grep 工具的 ENOENT 问题；现在回退到系统 `rg` 并在会话中途自愈。
- 修复 `/btw` 每次使用都把整个对话写盘的问题。
- 修复 `/context` Free space 和 Messages 明细与头部百分比不一致的问题。
- 修复多个插件问题：重复 `name:` frontmatter 时 slash 命令解析到错误插件、`/plugin update` 以 `ENAMETOOLONG` 失败、Discover 显示已安装插件、目录源插件从过期版本缓存加载、以及 skill 不遵循 `context: fork` 和 `agent` frontmatter 字段。
- 修复 `/mcp` 菜单对用 `headersHelper` 配置的 MCP server 提供 OAuth 专属操作的问题；现在改为提供 Reconnect 以重新调用 helper 脚本。
- 修复 `ctrl+]`、`ctrl+\` 和 `ctrl+^` 键位在发送原始 C0 控制字节的终端（Terminal.app、默认 iTerm2、xterm）中不触发的问题。
- 修复 `/login` OAuth URL 渲染带 padding、妨碍干净鼠标选择的问题。
- 修复渲染问题：非全屏模式下可见区上方内容变化时闪烁、非全屏长会话期间终端回滚被擦除、以及鼠标滚动转义序列偶尔泄漏进提示文本。
- 修复 `settings.json` 的 env 值为数字而非字符串时崩溃的问题。
- 修复应用内设置写入（如 `/add-dir --remember`、`/config`）未刷新内存快照、妨碍会话中途撤销已移除目录的问题。
- 修复 Bedrock、Vertex 等第三方 provider 上自定义键位（`~/.claude/keybindings.json`）不加载的问题。
- 修复 `claude --continue -p` 未正确继续由 `-p` 或 SDK 创建会话的问题。
- 修复多个 Remote Control 问题：会话崩溃时 worktree 被移除、连接失败未持久化到 transcript、本地会话 brief 模式下虚假「Disconnected」指示、以及仅设置 `CLAUDE_CODE_ORGANIZATION_UUID` 时 `/remote-control` 在 SSH 上失败。
- 修复 `/insights` 有时从响应中省略报告文件链接的问题。
- [VSCode] 修复关闭最后一个编辑器标签时聊天输入下方文件附件未清除的问题。

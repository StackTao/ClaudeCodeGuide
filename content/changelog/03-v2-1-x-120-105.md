## v2.1.x 系列（2.1.120 – 2.1.105）

### 2.1.120

**新功能**
- Windows：不再需要 Git for Windows（Git Bash）——缺失时 Claude Code 使用 PowerShell 作为 shell 工具。
- 新增 `claude ultrareview [target]` 子命令，可从 CI 或脚本非交互式运行 `/ultrareview`——把结论打到 stdout（`--json` 输出原始结果），完成时退出 0、失败时退出 1。
- Skills 现在可以在内容中用 `${CLAUDE_EFFORT}` 引用当前 effort 级别。
- 为子进程设置 `AI_AGENT` 环境变量，使 `gh` 能把流量归因到 Claude Code。

**改进**
- 当你已经安装了桌面 app 或已有 skills/agents 时，推荐安装它们的 spinner 提示现在会隐藏。
- 当终端发送方向键而非滚动事件时，显示「use PgUp/PgDn to scroll」提示。
- 配置了大量 claude.ai connector 但未授权时，会话启动更快。
- auto mode 拒绝消息现在链接到配置文档。
- `claude plugin validate` 现在接受 `marketplace.json` 顶层的 `$schema`、`version`、`description` 以及 `plugin.json` 中的 `$schema`。
- auto mode 中的 auto-compact 现在显示 `auto`（小写、无 token 计数）而非一个误导的 token 值。

**修复**
- 修复 stdio MCP 工具调用期间按 Esc 关闭整个 server 连接的问题（2.1.105 引入的回归）。
- 修复用 `claude --resume` 启动后 `/rewind` 和其他交互式覆盖层不响应键盘输入的问题。
- 修复非全屏模式下终端回滚重复的问题（缩放、关闭对话框、长会话）。
- 修复 `DISABLE_TELEMETRY` / `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` 未对 API 和企业用户抑制用量指标遥测的问题。
- 修复 auto mode 中对同时含管道和重定向的多行 bash 命令出现误报「Dangerous rm operation」权限提示的问题。
- 修复全屏模式下长选择菜单在终端下方被裁剪的问题——焦点选项现在随滚动保持在屏内。
- 修复全屏下点击「+N lines」时 Write 工具输出折叠而非展开的问题。
- 修复 slash 命令选择器在键入时跳动的问题，并改进高亮，使其只以蓝色匹配连续子串。
- 修复某个条目使用无法识别的源格式时 `/plugin` marketplace 加载失败的问题——该条目仍显示，但安装它会提示你更新。
- [VSCode] `/usage` 现在打开原生 Account & Usage 对话框，而不是返回纯文本会话成本。
- [VSCode] 语音听写现在遵循 `~/.claude/settings.json` 中的 `language` 设置。
- 修复 Bash 工具中的 `find` 在大目录树上耗尽打开文件描述符、导致主机范围崩溃的问题（macOS/Linux 原生构建）。

### 2.1.119

**新功能**
- 新增 `prUrlTemplate` 设置，把页脚 PR 徽标指向自定义代码审查 URL 而非 github.com。
- 新增 `CLAUDE_CODE_HIDE_CWD` 环境变量，在启动 logo 中隐藏工作目录。
- 新增：`--from-pr` 现在接受 GitLab merge-request、Bitbucket pull-request 和 GitHub Enterprise PR URL。

**改进**
- `/config` 设置（主题、编辑器模式、verbose 等）现在持久化到 `~/.claude/settings.json`，并参与 project/local/policy 覆盖优先级。
- `--print` 模式现在遵循 agent 的 `tools:` 和 `disallowedTools:` frontmatter，与交互模式行为一致。
- `--agent <name>` 现在对内置 agent 遵循 agent 定义的 `permissionMode`。
- PowerShell 工具命令现在可以在权限模式下被自动批准，与 Bash 行为一致。
- Hooks：`PostToolUse` 和 `PostToolUseFailure` hook 输入现在包含 `duration_ms`（工具执行时间，不含权限提示和 PreToolUse hook）。
- Subagent 和 SDK MCP server 重配置现在并行连接 server 而非串行。
- 被另一个插件版本约束固定的插件现在自动更新到满足条件的最高 git tag。
- Vim 模式：INSERT 中的 Esc 不再把排队消息拉回输入框；再按一次 Esc 打断。
- slash 命令建议现在高亮与你查询匹配的字符。
- slash 命令选择器现在把长描述换行到第二行而非截断。
- 输出中的 `owner/repo#N` 简写链接现在使用你 git remote 的主机而非总是指向 github.com。
- OpenTelemetry：`tool_result` 和 `tool_decision` 事件现在包含 `tool_use_id`；`tool_result` 还包含 `tool_input_size_bytes`。
- status line：stdin JSON 现在包含 `effort.level` 和 `thinking.enabled`。

**安全**
- `blockedMarketplaces` 现在正确强制 `hostPattern` 和 `pathPattern` 条目。

**修复**

### 2.1.118

**新功能**

- Hooks 现在可通过 `type: "mcp_tool"` 直接调用 MCP 工具。

- 新增 `claude plugin tag`，为插件创建发布 git tag 并带版本校验。

**改进**
- 把 `/cost` 和 `/stats` 合并进 `/usage`——两者作为键入快捷方式保留，会打开相应标签页。
- Auto mode：在 `autoMode.allow`、`autoMode.soft_deny` 或 `autoMode.environment` 中包含 `"$defaults"`，可在内置列表之外追加自定义规则而不是替换它。
- 为 auto mode opt-in 提示新增「Don't ask again」选项。
- `--continue`/`--resume` 现在能找到通过 `/add-dir` 添加了当前目录的会话。
- `/color` 现在在连接 Remote Control 时把会话强调色同步到 claude.ai/code。
- 使用自定义 `ANTHROPIC_BASE_URL` gateway 时 `/model` 选择器现在遵循 `ANTHROPIC_DEFAULT_*_MODEL_NAME`/`_DESCRIPTION` 覆盖。
- 当自动更新因另一插件的版本约束而跳过某插件时，跳过现在出现在 `/doctor` 和 `/plugin` Errors 标签页。

- 修复 `/fork` 每次 fork 都把完整父对话写盘的问题——现在写一个指针并在读取时填充。

### 2.1.117

**新功能**
- Forked subagent 现在可在外部构建上通过设置 `CLAUDE_CODE_FORK_SUBAGENT=1` 启用。
- 通过 `--agent` 的主线程 agent 会话现在会加载 agent frontmatter 的 `mcpServers`。

**改进**
- 改进 `/model`：即使项目固定了不同模型，选择现在也跨重启持久化，且启动头会显示活动模型何时来自项目或 managed-settings 固定。
- `/resume` 命令现在在重新读取过期、巨大的会话前提供先做摘要，与既有 `--resume` 行为一致。
- 同时配置本地和 claude.ai MCP server 时启动更快（并发连接现为默认）。
- 对已安装插件执行 `plugin install` 现在会安装任何缺失依赖，而不是停在「already installed」。
- 插件依赖错误现在显示「not installed」并带安装提示，且 `claude plugin marketplace add` 现在从已配置 marketplace 自动解析缺失依赖。
- managed-settings 的 `blockedMarketplaces` 和 `strictKnownMarketplaces` 现在在插件 install、update、refresh 和 autoupdate 时强制执行。
- Advisor Tool（实验性）：对话框现在带「experimental」标签、了解更多链接，启用时有启动通知；会话不再在每个提示和 `/compact` 上卡在「Advisor tool result content could not be processed」错误。
- `cleanupPeriodDays` 保留清理现在也覆盖 `~/.claude/tasks/`、`~/.claude/shell-snapshots/` 和 `~/.claude/backups/`。
- OpenTelemetry：`user_prompt` 事件现在为 slash 命令包含 `command_name` 和 `command_source`；模型支持 effort 级别时 `cost.usage`、`token.usage`、`api_request` 和 `api_error` 现在包含 `effort` 属性。除非设置 `OTEL_LOG_TOOL_DETAILS=1`，否则自定义/MCP 命令名会被脱敏。
- macOS 和 Linux 原生构建：`Glob` 和 `Grep` 工具被通过 Bash 工具可用的内嵌 `bfs` 和 `ugrep` 取代——无需单独工具往返即可更快搜索（Windows 和 npm 安装的构建不变）。
- Windows：按进程缓存 `where.exe` 可执行文件查找，加快子进程启动。
- Pro/Max 订阅者在 Opus 4.6 和 Sonnet 4.6 上的默认 effort 现在是 `high`（此前为 `medium`）。

**修复**
- 修复 Plain-CLI OAuth 会话在访问 token 中途过期时以「Please run /login」死亡的问题——token 现在在 401 时被反应式刷新。
- 修复 `WebFetch` 在极大 HTML 页面上挂起的问题——在 HTML 转 markdown 前截断输入。
- 修复代理返回 HTTP 204 No Content 时的崩溃——现在显示清晰错误而非 `TypeError`。
- 修复用 `CLAUDE_CODE_OAUTH_TOKEN` 环境变量启动且该 token 过期时 `/login` 无效的问题。
- 修复刚键入后提示输入撤销（`Ctrl+_`）无效、且每个撤销步骤跳过一个状态的问题。
- 修复在 Bun 下运行时远程 API 请求未遵循 `NO_PROXY` 的问题。
- 修复键名通过慢连接合并为文本到达时罕见的虚假 escape/return 触发。
- 修复 SDK `reload_plugins` 串行重连所有用户 MCP server 的问题。
- 修复 Bedrock application-inference-profile 请求在由禁用 thinking 的 Opus 4.7 支持时以 400 失败的问题。
- 修复 server 在回合中途完成连接时 print/SDK 模式下 MCP `elicitation/create` 请求自动取消的问题。
- 修复运行与主 agent 不同模型的 subagent 错误地用恶意软件警告标记文件读取的问题。
- 修复存在后台任务时的空闲重渲染循环，减少 Linux 上的内存增长。
- [VSCode] 修复配置多个大型 marketplace 时「Manage Plugins」面板崩溃的问题。
- 修复 Opus 4.7 会话显示虚高 `/context` 百分比、过早 autocompact 的问题——Claude Code 此前按 200K 上下文窗口而非 Opus 4.7 原生 1M 计算。

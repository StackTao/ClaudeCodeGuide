## 第十二章：最新功能与版本跟踪

Claude Code 迭代很快，指南必须区分“长期稳定原则”和“版本相关功能”。下面这些能力应定期复查官方文档：

| 功能 | 官方定位 | 使用建议 |
|---|---|---|
| `/goal` | 设置可验证完成条件，让 Claude 跨 turn 持续推进 | 只用于验收条件清楚的任务，写明停止边界 |
| Auto mode | 通过后台安全分类器减少权限提示 | 适合可信方向的低风险任务，不替代 review |
| Permission modes | 控制工具调用审批方式 | 团队项目优先默认/计划模式，高风险操作保持确认 |
| Plugins / Marketplace | 分发 skills、agents、hooks、MCP/LSP server | 官方 marketplace 优先，第三方必须 vet |
| Subagents | 独立上下文的专用 agent | 适合 review、test、security、debug 等专门任务 |
| Output styles | 调整 Claude 的系统行为风格 | 学习场景用 explanatory/learning，生产编码保持默认或团队风格 |
| Ultrareview | 云端多 agent 深度审查 | 重要 PR 合并前使用，成本和可用性需确认 |
| Ultraplan | 云端深度计划 | 跨模块大改造前使用，仍需人工审计划 |
| Claude Code SDK | 构建自定义 agent | 适合 CI bot、review bot、SRE/oncall 工具 |
| Routines / Schedule | 云端周期任务 | 适合定期 triage/report，不适合无人值守生产写操作 |
| Bundled skills | `/run`、`/verify`、`/code-review`、`/claude-api` 等官方内置技能 | 适合把运行验证、代码审查和 API 迁移变成可复用流程 |
| Opus 4.8 + `/effort` | v2.1.154 发布的最新模型，默认 high effort，`/effort xhigh` 应对最难任务 | 复杂架构/调试用 xhigh；简单任务降档省成本；Fast mode 在 4.8 上为 2x 价格换 2.5x 速度 |
| Dynamic workflows | v2.1.154 起让 Claude 在后台编排数十到上百个 agent，`/workflows` 查看 | 适合大规模审查/迁移/搜集；务必关注 token 成本 |

版本跟踪建议：

- 每月检查一次官方 `code.claude.com/docs` 和 Anthropic docs。
- 新命令先写入“待验证”区，至少找到官方文档或 GitHub 仓库后再升为正式建议。
- X、LinuxDO、Reddit 只作为发现线索，不能作为命令事实的唯一依据。
- 对 preview/research preview 功能标注可用性、成本、账号要求和风险。

本章要点：

- Claude Code 的命令和插件市场变化很快。
- 官方功能事实和社区技巧要分层。
- 自动化越强，越要写清适用边界。

实战清单：

- 记录每个新功能的官方链接。
- 对 preview 功能写明版本、计划和费用限制。
- 定期删除过时命令和失效安装方式。

### 12.1 v2.1.144 / v2.1.145 快照（2026-05-19 至 05-20）

这两个版本是最近一次密集更新，跟踪文档时优先校对以下变化：

| 变化 | 说明 | 实操影响 |
|---|---|---|
| `/resume` 支持 bg 会话 | 后台 (`claude --bg`) 会话现在出现在 `/resume` 列表，标记 `bg` | 不再需要单独记后台 session id |
| `/model` 默认行为（已于 v2.1.153 再次调整，见 12.3） | v2.1.144 一度改为默认仅本次会话、按 `d` 设默认 | 此说法已过时，最新行为见 12.3 |
| `/extra-usage` → `/usage-credits` | 命令重命名；旧名称仍兼容 | 文档与团队脚本同步更新 |
| `claude agents --json` | 以 JSON 列出所有活动会话 | 适合接入 tmux-resurrect、状态栏、会话选择器 |
| OTEL 增强 | `claude_code.tool` span 增加 `agent_id`、`parent_agent_id`，subagent span 正确嵌套到父 Agent | 追踪并行 agent 调用链 |
| Read tool 截断回退 | 超 token 限时返回带 “PARTIAL view” 提示的截断首页，不再硬失败 | 大文件读取更稳，但要警惕只读到前半段 |
| 安全修复 | 对非白名单环境变量的裸 `VAR=value` 赋值需显式批准 | 之前可能绕过 permission 提示的写法被堵 |
| `/plugin` Discover/Browse 预览 | 安装前可看 plugin 暴露的 commands / agents / skills / hooks / MCP / LSP servers，以及预估 context 成本 | 第三方插件 vet 流程因此可量化 |
| `claude plugin disable/enable` | 依赖强制：禁用被依赖的插件会拒绝并提示完整禁用链；`enable` 强制启用传递依赖 | 团队脚本批量启停需考虑依赖关系 |
| 状态栏 JSON 输入 | 检测到 GitHub 仓库时自动注入 repo / PR 信息 | 自定义 statusline 可直接读取 |
| API 不可达启动 | 启动握手超时从 75s 降为 15s | 离线/网络抖动时不会卡住 |
| Windows 后台会话滚动修复 | PgUp/PgDn、滚轮、`Ctrl+O` transcript 重新可用 | Windows 用户终于可以 attach 后台会话滚屏 |

来源：官方 [changelog](https://code.claude.com/docs/en/changelog)、`anthropics/claude-code` 仓库 CHANGELOG.md、Releasebot 整理稿。

### 12.3 v2.1.146 → v2.1.156 快照（2026-05-21 至 05-29）

以官方 [changelog](https://code.claude.com/docs/en/changelog) 核验，重点变化如下。

**最重磅：Opus 4.8（v2.1.154，2026-05-28）**

- Opus 4.8 发布，模型 ID `claude-opus-4-8`，默认就是 high effort，`/effort xhigh` 用于最难的任务。
- Fast mode 在 Opus 4.8 上以「2x 标准价换 2.5x 速度」提供，且仍是 Opus 本体而非降级小模型。
- 除 Haiku、Sonnet 与 Opus 4.7 及更早模型外，**精简版系统提示（lean system prompt）成为默认**。
- v2.1.156（2026-05-29）修复了 Opus 4.8 下 thinking block 被改写导致 API 报错的问题——用 4.8 的话务必升到 ≥ v2.1.156。

**Dynamic workflows（v2.1.154）**

- 让 Claude 自行编排一个工作流，在后台跨数十到上百个 agent 推进，`/workflows` 查看运行情况。
- 适合大规模审查、迁移、情报搜集这类「一个上下文装不下」的任务；代价是 token 消耗可能很大，需要用户明确选择。

**代码审查命令的最终形态（务必注意，几个版本反复横跳）**

| 命令 | 当前（≥ v2.1.154）职责 | 演化 |
|---|---|---|
| `/code-review [effort] [--fix] [--comment] [target]` | 找 correctness bug；`--fix` 把结论应用到工作区，`--comment` 发成 GitHub PR inline 评论 | v2.1.147 由 `/simplify` 改名而来 |
| `/simplify` | **清理向审查**：复用、简化、效率、抽象层级（altitude），并应用修复 | v2.1.152 一度等于 `/code-review --fix`，v2.1.154 又独立回清理职责 |

所以现在 `/simplify` 和 `/code-review` 是**两个不同命令**，不要再当作别名。

**后台会话与 agents（v2.1.147 / v2.1.153 / v2.1.154）**

- Pinned background sessions：在 `claude agents` 里 `Ctrl+T` 固定，空闲不回收、更新时原地重启，仅在内存压力下且非 pinned 会话先被回收。
- `claude agents` 里输入 `! <command>` 可把 shell 命令作为可 attach/detach 的后台会话运行；也可 `claude --bg --exec '<command>'`。
- `claude agents` dispatch 自动补全会建议原生 slash 命令和内置 skills；PR 列显示 `PR #N` / `N PRs`。

**Hooks / Skills / 设置（v2.1.149 / v2.1.152）**

- 新增 `/reload-skills` 命令，不重启即可重新扫描 skill 目录；`SessionStart` hook 可返回 `reloadSkills: true`，也可用 `hookSpecificOutput.sessionTitle` 设会话标题。
- 新增 `MessageDisplay` hook 事件，可改写/隐藏助手输出文本。
- skills 和 slash 命令的 frontmatter 支持 `disallowed-tools`。
- 新增 `pluginSuggestionMarketplaces`（允许组织 marketplace 进入建议）与 `allowAllClaudeAiMcps`（加载 claude.ai 云端 MCP connector）等托管设置。
- **Auto mode 不再需要事先 opt-in 同意**（v2.1.152）；AskUserQuestion 在 auto mode 下只在 Claude 确实无法自行决定时弹出（v2.1.154）。
- 主模型找不到时，Claude Code 会在本会话剩余时间切到 `--fallback-model`。

**安全修复（v2.1.149，值得专门留意）**

- 修复 PowerShell 权限绕过：内置 `cd` 变体（`cd..`、`cd\`、`cd~`、`X:`）可在未被检测的情况下切目录。
- 修复 git worktree 下 sandbox 写白名单覆盖到整个主仓库根目录。
- 修复权限分析信任了过期的 `PWD`/`OLDPWD`/`DIRSTACK`。
- v2.1.154 另修了 `HOME` 带尾斜杠时 `rm -rf $HOME` 未被拦截、以及 sandbox 内外 `$TMPDIR` 解析不一致。

**用量与可观测（v2.1.149 / v2.1.153）**

- `/usage` 现在按类别拆分（skills、subagents、plugins、各 MCP server 成本）。
- `/diff` 详情可键盘滚动；Markdown 输出渲染 GFM 任务清单复选框。
- status line 命令现在能拿到 `COLUMNS` 和 `LINES` 环境变量；`claude doctor` 会显示上次更新尝试的结果。
- `github`/`git` 插件 marketplace 源支持 `skipLfs` 跳过 Git LFS 下载。

实操建议：

- 用 Opus 4.8 的，升到 ≥ v2.1.156 再开始重要任务，避开 thinking block 报错。
- 团队若依赖 `/simplify` 做清理，确认大家理解它已与 `/code-review` 分家。
- 仍在 v2.1.148 的尽快升级——该版仅为修 v2.1.147 引入的 Bash 退出码 127 回归。

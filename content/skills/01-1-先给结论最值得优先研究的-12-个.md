## 1. 先给结论：最值得优先研究的 12 个

这张表是“研究和试用优先级”，不是安装清单。官方基础设施可以优先上生产项目；第三方插件和社区 skill 先在个人或隔离项目验证，确认 hooks、MCP、LSP、shell 命令和上下文成本后再进团队配置。

| 优先级 | 名称 | 来源等级 | 解决的问题 | 适合谁 |
|---|---|---|---|---|
| S | 语言 LSP 插件 | 官方事实 | 跳转定义、查引用、类型诊断 | 所有代码项目 |
| S | GitHub 插件 / GitHub Action | 官方事实 | issue、PR、CI、review 工作流 | GitHub 项目 |
| S | 官方 Commands / bundled skills | 官方事实 | `/code-review`、`/run`、`/verify`、`/claude-api` 等内置流程 | 所有项目先掌握 |
| A | Superpowers | 官方插件页 + 社区双源验证 | brainstorming、计划、TDD、subagent review、系统调试流程 | 严肃软件开发、复杂功能、团队工程 |
| A | Code Simplifier | 官方插件页 / 社区线索 | 简化最近改动、反过度设计 | AI 代码清理 |
| A | Context7 | 社区双源验证 | 拉取当前库文档，减少 API 幻觉 | 前端、后端、框架更新快的项目 |
| A | GSD / Get Shit Done | 社区双源验证 | 上下文隔离、spec-driven、wave execution | 大任务、长会话、重自动化 |
| A | OpenAI Codex Plugin | 社区双源验证 | second pass、adversarial review、rescue | 重要改动复核 |
| B | last30days | 社区线索 | 搜索最近 30 天 Reddit/X/HN/YouTube/web 热点 | 调研、选型、市场/AI 生态跟踪 |
| B | document-skills | Anthropic skills / 插件生态 | PDF、DOCX、PPTX、XLSX 等办公文档处理 | 文档、研究、运营 |
| B | Trail of Bits security skills | 活跃安全仓库 | 安全审计、漏洞研究、代码安全检查 | 安全敏感项目 |
| B | Figma / Frontend Design / Browser QA | 官方/社区组合 | 设计稿到实现、视觉验证 | 前端项目 |

我的建议：

- **先掌握内置能力**：官方 Commands、bundled skills、LSP、GitHub。
- **再按场景试用第三方**：Superpowers、Context7、Code Simplifier、GSD。
- **高风险改动再加复核**：Codex plugin、Trail of Bits security skills、人类 review。
- **调研类工具隔离使用**：last30days 这类会访问外部内容的 skill，不写入生产项目默认上下文。

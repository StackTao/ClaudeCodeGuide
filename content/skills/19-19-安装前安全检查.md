## 19. 安装前安全检查

来源等级：官方插件安全边界 + 通用供应链审查。Claude Code 插件详情页会显示 context cost、last updated、Will install 和来源信息，但这些信息不能替代人工审查。

安装任何第三方 plugin/skill 前检查：

- 来源是否是官方 marketplace、知名 GitHub、还是陌生仓库。
- 是否有 `.claude-plugin/plugin.json`。
- `Will install` 里包含哪些 skills、agents、hooks、MCP/LSP servers。
- 是否执行 shell 命令。
- 是否有 hooks，尤其是 `PreToolUse` 和 `PostToolUse`。
- 是否读取 `.env`、home、SSH key、token。
- 是否联网下载脚本。
- 是否混淆代码或 base64 大块。
- 是否近期维护。
- 是否有 issue 报告安装失败或自动触发不稳定。

安装范围：

| 范围 | 用途 | 建议 |
|---|---|---|
| User | 个人长期工具 | 最常用 |
| Local | 某项目个人实验 | 试用推荐 |
| Project | 团队共享 | 需要 review |
| Managed | 公司统一 | 管理员维护 |

最低验证流程：

1. 先看官方 marketplace 是否已有同类插件。
2. 再看插件来源、manifest、Will install、context cost、更新时间。
3. 然后在临时项目安装，观察它是否自动触发、是否读写文件、是否联网。
4. 最后决定安装范围：个人 user、项目 local、团队 project，或放弃。

禁止默认放行：

- 未说明来源的大包。
- 安装脚本下载并执行远程代码。
- hooks 自动阻止或改写敏感文件。
- 要求 broad token、SSH key、生产云账号写权限。

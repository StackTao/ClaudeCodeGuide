## 16. 官方 LSP 插件：最稳的基础设施

来源等级：官方插件 / 官方 marketplace。语言列表和依赖名称仍要以当前插件详情页为准。

如果项目是代码仓库，语言 LSP 插件几乎是第一优先级。

| 语言 | 插件 | 依赖 |
|---|---|---|
| TypeScript | `typescript-lsp` | `typescript-language-server` |
| Python | `pyright-lsp` | `pyright-langserver` |
| Go | `gopls-lsp` | `gopls` |
| Rust | `rust-analyzer-lsp` | `rust-analyzer` |
| Java | `jdtls-lsp` | `jdtls` |
| C/C++ | `clangd-lsp` | `clangd` |
| C# | `csharp-lsp` | `csharp-ls` |
| Kotlin | `kotlin-lsp` | `kotlin-language-server` |
| PHP | `php-lsp` | `intelephense` |
| Swift | `swift-lsp` | `sourcekit-lsp` |

安装：

```text
/plugin install typescript-lsp@claude-plugins-official
/reload-plugins
```

使用边界：

- LSP 提供符号、诊断和跳转能力，不保证 Claude 的修改一定正确。
- 多语言 monorepo 只启用本轮任务需要的 LSP，避免上下文和启动成本过高。
- 依赖安装失败时先按语言服务器自身文档排错，再判断是否是 Claude Code 插件问题。

验证方式：安装后让 Claude 查询一个真实 symbol 的定义和引用，再跑项目原有类型检查或测试。

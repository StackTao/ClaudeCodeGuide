## 11. last30days：追踪最近 30 天社区情报

来源等级：社区线索 / 中低。last30days 是社区传播的 research skill；安装方式和数据源范围必须以当前仓库 README 或 marketplace manifest 为准。

核心结论：它适合发现最近社区在讨论什么，不适合作为事实来源本身。

安装线索：

```text
/plugin marketplace add mvanhorn/last30days-skill
/plugin install last30days@last30days-skill
/reload-plugins
```

或：

```text
claude install-skill https://github.com/mvanhorn/last30days-skill
```

适合：

- 查 Claude Code 新命令、新插件。
- 查某个技术栈最近坑点。
- 查 AI 工具生态趋势。
- 做市场/内容研究。

注意：

- 它会访问大量外部内容，可能慢且消耗高。
- 结果应作为线索，核心事实还要回到官方文档/GitHub。
- X、Reddit、HN 里的结论容易缺版本和环境，不要直接写入团队规范。
- 涉及公司、客户或未公开项目名时，先确认搜索内容是否可外发。

验证方式：

- 对每条结论找官方文档、仓库 issue、release note 或第二个独立来源。
- 无法验证的内容只写成“社区线索”，不要升格为安装或命令事实。

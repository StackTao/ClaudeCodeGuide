## 11. 全站审校清单

> 审校日期：2026-05-25  
> 目标：把主指南、命令手册、skills 文档、模板和来源索引统一到“官方事实准确、社区经验可验证、中文教程清晰”的标准。

### 11.1 官方已确认

| 主题 | 已对齐来源 | 当前处理 |
|---|---|---|
| Commands reference | https://code.claude.com/docs/en/commands | 命令手册以官方命令表为基线，补入 `/autofix-pr`、`/code-review`、`/copy`、`/claude-api`、`/voice` |
| CLI reference | https://code.claude.com/docs/en/cli-usage | CLI 章节继续以官方 flags 和启动方式为准 |
| Changelog | https://code.claude.com/docs/en/changelog | 版本相关内容只在“最新功能”章节写明日期和版本 |
| Prompt Library | https://code.claude.com/docs/en/prompt-library | 模板页保留中英对照改写，不逐字镜像官方内容 |
| Permissions / auto mode / settings | 官方 permissions、auto mode、settings 文档 | 安全和权限章节保留“默认最小权限、高风险确认”的原则 |

### 11.2 需要持续复查

| 区域 | 风险 | 下一步 |
|---|---|---|
| Slash commands | 命令迭代快，社区文章容易滞后 | 每月用官方 Commands 页和本机 `/help` 对照 |
| Plugins / Marketplace | 第三方 marketplace 变化快，安装方式可能失效 | 官方 marketplace 优先，第三方先查 manifest、hooks、MCP、权限 |
| Skills 推荐 | 社区插件体验差异大 | 保留适用场景和风险，不写成“必装”绝对结论 |
| FAQ | Windows、代理、网络、证书问题受环境影响大 | 官方 troubleshooting 优先，社区内容标注版本和环境 |
| X / Twitter 线索 | 登录限制、短帖缺上下文 | 只作为发现入口，不能作为命令事实 |

### 11.3 建议降级为社区线索的内容

| 内容类型 | 处理规则 |
|---|---|
| 单个 X trend 或短帖总结的新命令 | 不写入命令正文；如果官方 Commands 页没有对应条目，标为待验证 |
| 单个 LinuxDO / CSDN 排错帖 | 只放 FAQ 排查方向，不写成通用解决方案 |
| “效率提升 N 倍”“必装插件”类结论 | 改为适用场景、前置条件和风险说明 |
| 第三方大包插件推荐 | 强调先在隔离项目试用，检查 hooks、MCP、工具权限和上下文成本 |

### 11.4 样式统一标准

- 每个面向用户的章节优先给“适用场景、核心结论、风险边界、验证方式、来源等级”。
- 长段落拆成短段落或表格；教程保留背景，但避免堆砌故事化描述。
- 命令事实写具体命令和官方来源；社区经验写环境、版本和可信度。
- 不改变已有主标题，避免破坏已分享的 URL fragment。
